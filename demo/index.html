Exam Integrity Model

Three mechanisms for a custody chain that can't stay diffused.

Most exam-security reforms assume theft in transit: sealed containers, armed transport, restricted printing. They do little against a different threat model, someone who already has legitimate access. These three mechanisms are aimed at that threat model specifically. All three run live below, this is not a mockup.

01 — Custody chain

Diffused, until it isn't.

A custody log where obligations aren't fixed on a named person is, in practice, unaccountable: no one is individually traceable, so nothing about a handoff can be verified after the fact. Toggle the chain below. In pinned mode, every hash is computed in this page, right now, with the browser's native crypto.subtle.digest, nothing is precomputed.

Six people touch this document. In this model, none of them are individually traceable.

Edit the Transport Custodian's logged note, then verify. Each signature equals SHA-256(previous hash + role + note + timestamp), so changing one field breaks that hash, and every hash chained after it.

Roles and notes are illustrative placeholders. The hashing itself is genuine SHA-256, not simulated.

02 — Secret sharing

Nothing whole to leak.

If a full document, or the key that unlocks it, exists in one place, it can be copied by one person. Below, a real secret is split across six custodians using Shamir's Secret Sharing over GF(256). Summary

Three mechanisms, one supply-chain problem.

A custody chain that can't stay diffused. A secret with nothing whole to steal until enough custodians agree. Detection that doesn't require manual inspection. Each one runs live on this page, and each one is backed by a test in the repository.

View the repository → Read the documentation
What actually runs live on this page:
  • Module 01 computes a genuine SHA-256 hash chain with the browser's native crypto.subtle, nothing precomputed.
  • Module 02 runs a real 4-of-6 Shamir's Secret Sharing scheme over GF(256), reconstructing, or failing to, byte for byte.
  • Module 03 computes a real Jaccard shingle-similarity score on whatever text you type.
None of this is connected to any real institution's systems or data. See the project README for full scope and limitations.
